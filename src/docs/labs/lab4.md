# Лабораторная работа №4
## Тема: Предсказание дефолта по кредиту

### 1. Цель
Построить модели классификации для оценки кредитного риска и сравнить их качество с помощью метрик Accuracy и ROC-AUC.

### 2. Ход работы
1.  Загружены данные Kaggle "Give Me Some Credit".
2.  Пропуски в `MonthlyIncome` и `NumberOfDependents` заполнены средними значениями из обучающей выборки.
3.  Обучены модели: Logistic Regression, Random Forest и CatBoost (самостоятельная работа).
4.  Оценено качество на тестовой выборке.

### 3. Результаты
Сравнение моделей показало преимущество градиентного бустинга. Метрика ROC-AUC (площадь под кривой ошибок) для различных алгоритмов:

| Модель | ROC-AUC |
| :--- | :--- |
| Logistic Regression | 0.69 |
| Random Forest | 0.84 |
| **CatBoost** | **0.86** |

*Примечание: Метрика Accuracy (~93%) близка к доле класса "без дефолта" в выборке, поэтому для оценки использовался ROC-AUC, который лучше отражает способность модели выявлять редкие события (дефолты).*

### 4. Интеграция с веб-сервисом
Для внедрения модели в банковский процесс предлагается использовать стек **Python + FastAPI**:
1.  Сохранение обученной модели в файл (`joblib`).
2.  Создание API-эндпоинта `/predict`, принимающего данные клиента в JSON.
3.  Применение той же предобработки (заполнение пропусков средним), что и при обучении.
4.  Возврат вероятности дефолта клиенту.
